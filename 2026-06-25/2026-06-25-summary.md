## 1. Tmax: A Simple Recipe for Terminal Agents

- **arXiv**: [2606.23321](https://arxiv.org/abs/2606.23321)
- **Authors**: Hamish Ivison, Junjie Oscar Yin, Rulin Shao et al. (Allen AI / UW)
- **方向**: 智能体、推理能力增强
- **热度**: 140 Bookmarks

**Problem**: 终端环境中的复杂长序列任务对 LLM 智能体提出了高要求，现有开源模型在 Terminal-Bench 上表现不佳，缺乏有效的合成数据管线和训练方案。

**Method**: 提出 TMAX-15K 合成数据管线，通过组合式难度控制、角色多样化和分级验证器生成 14,600 个 RL 环境实例。训练采用简洁的 RL recipe（DPPO + FP32 LM head），无需复杂工程技巧即可训练出高性能终端智能体。

**Key Results**: TMAX-9B 在 Terminal-Bench 2.0 上达到 27%，是 10B 以下参数规模的开源 SOTA；同时泛化到 SWE-Bench 和 AIME 等下游任务，展现了数据管线和训练方案的通用性。

**Conclusion**: 高质量的合成 RL 数据和简洁的训练 recipe 足以训练出强终端智能体，无需依赖复杂架构或大规模人工标注。

## 2. Unlimited OCR Works

- **arXiv**: [2606.23050](https://arxiv.org/abs/2606.23050)
- **Authors**: Youyang Yin, Huanhuan Liu, Qunyi Xie et al. (Baidu)
- **方向**: 高效推理、多模态
- **热度**: 52 Bookmarks

**Problem**: 现有 OCR 模型难以处理长周期多页文档解析，KV cache 随页面数量线性增长导致计算成本不可控。

**Method**: 提出 Reference Sliding Window Attention（R-SWA），将解码端注意力约束为"全局视觉 token + 局部滑动窗口（128 tokens）"，保持 KV cache 恒定大小。支持单次前向传播完成多页文档的端到端解析。

**Key Results**: 在 OmniDocBench v1.5 上达到 93.23%（比 DeepSeek OCR 提升 6.22%）；首次实现单次前向传播解析 40+ 页文档，计算效率显著提升。

**Conclusion**: R-SWA 机制在不牺牲精度的前提下实现了恒定的 KV cache 开销，使长文档端到端 OCR 解析成为可能。

## 3. Qwen-AgentWorld: Language World Models for General Agents

- **arXiv**: [2606.24597](https://arxiv.org/abs/2606.24597)
- **Authors**: Yuxin Zuo, Zikai Xiao, Li Sheng et al. (Qwen Team)
- **方向**: 智能体、推理能力增强、评估基准
- **热度**: 34 Bookmarks

**Problem**: 现有智能体训练依赖真实环境交互，成本高且可控性差；缺乏能覆盖多领域环境的语言世界模型来支持可控的 SimRL 训练。

**Method**: 提出首个覆盖 7 个领域的原生语言世界模型（LWM），采用三阶段训练（CPT 注入 → SFT 激活 → RL 锐化），在 10M+ 环境轨迹上训练。支持两大范式：作为环境模拟器支持可控 SimRL；作为智能体基础模型通过"下一状态预测"进行元推理。

**Key Results**: 397B 版本在 AgentWorldBench 上超越 GPT-5.4；作为 SimRL 环境训练的智能体超越真实环境训练效果；"下一状态预测"元推理能力提升 7 个下游基准表现。

**Conclusion**: 语言世界模型可以同时充当环境模拟器和智能体基础模型，为通用智能体训练提供了可扩展的新范式。

## 4. Tapered Language Models

- **arXiv**: [2606.23670](https://arxiv.org/abs/2606.23670)
- **Authors**: Reza Bayat, Ali Behrouz, Aaron Courville et al.
- **方向**: 高效推理
- **热度**: 29 Bookmarks

**Problem**: 标准 Transformer 语言模型在所有层使用相同的 MLP 宽度，但研究表明浅层和深层对宽度的依赖程度不同，存在参数冗余。

**Method**: 提出 Tapered Language Models（TLM），沿深度方向单调递减 MLP 宽度（采用余弦衰减曲线），在保持总参数量不变的情况下重新分配计算资源，使浅层获得更多参数而深层适当缩减。

**Key Results**: 零额外训练成本即可提升 perplexity 表现；在相同参数量预算下，TLM 架构优于等宽度的标准 Transformer。

**Conclusion**: 语言模型不需要等宽度的层结构，沿深度递减宽度的锥形架构更高效，为模型架构设计提供了新的自由度。

## 6. HoloAgent-0: A Unified Embodied Agent Framework with 3D Spatial Memory

- **arXiv**: [2606.23565](https://arxiv.org/abs/2606.23565)
- **Authors**: Xiaolin Zhou, Liu Liu, Tingyang Xiao et al. (Horizon Robotics / D-Robotics)
- **方向**: 智能体、多模态
- **热度**: 22 Bookmarks

**Problem**: 现有具身智能体框架缺乏统一的 3D 空间感知和记忆机制，导致在复杂室内环境中的导航和任务执行能力受限。

**Method**: 提出统一的具身智能体框架 HoloAgent-0，集成 AgentOS（操作系统层）、3D 空间记忆（场景表示与检索）和技能层（动作原语与组合策略），实现感知-记忆-行动的闭环。

**Key Results**: 在多个具身智能基准上展示了优于现有框架的任务完成率和泛化能力，3D 空间记忆显著提升了长序列任务表现。

**Conclusion**: 统一的具身智能框架（OS + 3D 记忆 + 技能层）是构建通用具身智能体的有效路径。

## 7. Agent-as-a-Router: Agentic Model Routing for Coding Tasks

- **arXiv**: [2606.22902](https://arxiv.org/abs/2606.22902)
- **Authors**: Pengfei Zhou, Zhiwei Tang, Yixing Ma et al.
- **方向**: 智能体、评估基准
- **热度**: 21 Bookmarks

**Problem**: 代码任务中，不同模型在不同类型问题上各有优劣，静态选择单一模型无法最大化整体性能。

**Method**: 提出 Agent-as-a-Router 框架，采用 C-A-F（Classify-Analyze-Forward）闭环机制，动态分析任务特征并路由到最合适的模型。研究发现信息缺失是路由性能的核心瓶颈。

**Key Results**: 累积遗憾（cumulative regret）在所有对比方法中最低；结构化通信协议和较小团队规模的路由效果最优。

**Conclusion**: 动态路由框架能有效提升代码任务中多模型协作的整体性能，信息完整性和通信协议设计是路由成功的关键。

## 8. MAS-PromptBench: When Does Prompt Optimization Improve Multi-Agent LLM Systems?

- **arXiv**: [2606.23664](https://arxiv.org/abs/2606.23664)
- **Authors**: Juyang Bai, Laixi Shi et al. (JHU)
- **方向**: 智能体、评估基准
- **热度**: 20 Bookmarks

**Problem**: Prompt 优化在多智能体 LLM 系统中的效果不稳定，缺乏系统性的研究来明确何时 prompt 优化能带来正向收益。

**Method**: 构建 MAS-PromptBench 基准，系统量化 prompt 优化在不同任务类型、工作流架构、通信协议和团队规模下的效果。覆盖多种 prompt 优化算法和多智能体配置。

**Key Results**: Prompt 优化的效果范围从 +24 到 -16 不等，取决于具体配置；结构化通信和较小团队规模下收益最大；某些场景下 prompt 优化反而有害。

**Conclusion**: Prompt 优化在多智能体系统中并非总是有益的，需要根据具体配置审慎选择，该基准为后续研究提供了系统化的评估框架。

## 9. SeFi-Image: A Text-to-Image Foundation Model with Semantic-First Diffusion

- **arXiv**: [2606.22568](https://arxiv.org/abs/2606.22568)
- **Authors**: SeFi-Team
- **方向**: 多模态、高效推理
- **热度**: 16 Bookmarks

**Problem**: 现有文生图模型在语义理解和纹理生成之间缺乏有效解耦，导致训练效率低且语义一致性差。

**Method**: 提出 Semantic-First Diffusion（SFD），将语义理解和纹理生成分离为两个阶段。先建立语义骨架，再填充纹理细节。5B 参数模型仅需 125K A800 GPU 小时训练（约为 Z-Image 的 10-20%）。

**Key Results**: GenEval 达到 0.88、LongTextBench 达到 0.978、OneIG 达到 0.561，匹敌或超越 Qwen-Image 和 Z-Image；训练效率提升 5-10 倍。

**Conclusion**: 语义优先的扩散策略能显著提升文生图模型的训练效率和生成质量，语义-纹理解耦是有效的设计范式。

## 10. MeshFlow: Mesh Generation with Equivariant Flow Matching

- **arXiv**: [2606.23489](https://arxiv.org/abs/2606.23489)
- **Authors**: Qi Sun, Kiyohiro Nakayama, Jing Nathan Yan et al. (CityUHK / Stanford / Cornell / UT Austin)
- **方向**: 多模态
- **热度**: 15 Bookmarks

**Problem**: 现有 3D mesh 生成方法要么依赖自回归解码（速度慢），要么缺乏对 mesh 几何对称性的建模（质量差）。

**Method**: 提出首个等变流匹配（Equivariant Flow Matching）模型用于直接生成三角形 mesh。采用对称性感知 DiT 架构和嵌套最优传输耦合，在保持几何等变性的同时实现高效采样。

**Key Results**: Mesh 质量达到 SOTA 水平；推理速度 0.877s，比自回归方法（约 16s）快 18 倍。

**Conclusion**: 等变流匹配结合了流匹配的高效采样和几何等变性的先验知识，是 3D mesh 直接生成的有效方法。
